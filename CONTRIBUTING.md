# Contributing

## Reporting Issues

Check existing issues before filing a new one. Include:

- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, version, language runtime)

## Feature Requests

Open an issue describing the problem you want to solve and your proposed approach.

## Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-change`)
3. Make your changes following the project's code style
4. Add or update tests as needed
5. Run tests to confirm nothing is broken
6. Commit with a descriptive message
7. Open a pull request against the main branch

## Code Style

- Follow existing patterns in the codebase
- Use consistent naming conventions
- Write clear comments explaining why, not what
- Keep functions focused and small

## Testing

- Add tests for new functionality
- Ensure existing tests pass before submitting
- Test edge cases and error conditions
